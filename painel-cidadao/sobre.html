Sobre e Metodologia

O Fiscaliza Varginha é uma ferramenta independente de fiscalização cidadã que agrega dados públicos da Prefeitura e da Câmara Municipal de Varginha-MG em um painel único, legível e auditável.

⚠️ Aviso legal importante

Este painel não acusa ninguém. Os dados são extraídos de fontes oficiais e apresentados como estão, com alertas automáticos baseados em critérios documentais (ausência de objeto claro, falta de data, valor zerado). Um alerta não é prova de irregularidade — é um ponto de partida para fiscalização com fontes primárias.

Antes de divulgar qualquer dado, consulte o documento original no Portal Betha ou solicite via LAI (e-SIC). Ferramenta sem fins lucrativos, sem vínculo partidário.

🎯 Objetivo

Facilitar a fiscalização cidadã colocando dados de contratos, licitações, diárias, emendas, folha de pagamento e aluguéis em um único painel, com alertas automáticos e links diretos para as fontes oficiais.

Qualquer cidadão, jornalista, vereador ou servidor pode usar livremente.

🔄 Atualização dos dados

Os dados são coletados periodicamente por um script Python (coletor.py) que consulta a API REST do Portal Betha e o SAPL da Câmara.

  • Contratos e licitações: semanalmente
  • Emendas: mensalmente
  • Folha de pagamento: conforme disponibilidade
  • Diárias: conforme disponibilidade

⚙️ Auditômetro

Cada contrato recebe uma pontuação de 0 a 100 baseada em critérios documentais:

  • Objeto com descrição clara (mín. 25 caracteres)
  • Valor preenchido e positivo
  • Data de assinatura presente
  • Data de término preenchida

Não avalia legalidade — apenas completude documental.

⚠️ Cruzamento CNPJ

Quando uma empresa contratada pela Prefeitura também recebeu emenda impositiva de algum vereador, o painel exibe um alerta no card do contrato.

O cruzamento usa os 8 primeiros dígitos do CNPJ (raiz), pois o portal mascara os últimos 4 dígitos por LGPD. Um cruzamento positivo pede verificação — não indica irregularidade por si só.

Como os dados chegam ao painel

  1. 1
    Coleta automática Script Python consulta APIs REST do Portal Betha e SAPL. Salva os dados em arquivos JSON locais.
  2. 2
    Geração do bundle Um segundo script compila os JSONs num único arquivo data.js (variável global ZELA_DATA).
  3. 3
    Renderização no browser app.js lê ZELA_DATA, aplica os filtros, alertas e formatações e preenche as páginas HTML com JavaScript puro — sem servidor.
  4. 4
    Cache local (Service Worker) Na segunda visita, o browser usa a versão em cache. O painel funciona offline com os dados da última coleta.

Limitações conhecidas

  • CNPJ mascarado: o portal oculta os últimos 4 dígitos por LGPD. O cruzamento usa apenas os 8 primeiros.
  • Dados históricos: a API Betha retorna dados a partir do ano configurado. Anos anteriores podem estar incompletos.
  • Folha de pagamento: nomes e cargos refletem a competência mais recente disponível.
  • Defasagem: pode haver lag de alguns dias entre o ato oficial e a publicação no portal.
  • Correspondência de emendas: algumas emendas não têm CNPJ registrado e não entram no cruzamento.
  • Sem acesso a documentos físicos: o painel não substitui o pedido LAI para documentos completos.
